## 🎯 О проекте

SmartSupport — это инновационная система обработки обращений граждан, которая использует возможности искусственного интеллекта для:

- **Автоматического анализа** обращений (категория, приоритет, тональность)
- **Интеллектуального поиска** релевантной информации в базе знаний
- **Генерации человекоподобных ответов** для операторов
- **Масштабируемой обработки** обращений в режиме реального времени

### Цель проекта

Создать эффективную систему, которая автоматически анализирует обращения граждан, находит релевантную информацию в базе знаний и предоставляет операторам готовые варианты ответов, значительно ускоряя процесс обработки запросов.

### Статус

**MVP (Minimum Viable Product)** - реализован минимальный функционал для демонстрации основных возможностей системы. Тестовая база знаний содержит 9 статей по различным категориям для демонстрации работы интеллектуального поиска и генерации ответов.

## ✨ Возможности

### Для граждан

- 📝 **Простая форма подачи обращений** с автоматической категоризацией
- 💬 **Чат в реальном времени** с операторами (WebSocket)
- 📱 **Telegram бот** для подачи обращений и общения
- 🔍 **Отслеживание статуса** обращения по номеру
- 📱 **Адаптивный интерфейс** для всех устройств

### Для операторов

- 🎯 **Панель управления обращениями** с фильтрацией и сортировкой
- 🤖 **ИИ-подсказки** с готовыми вариантами ответов
- 💬 **Чат-интерфейс** для общения с гражданами (Web + Telegram)
- 📊 **Информация о приоритете и тональности** каждого обращения
- 🔄 **Умное управление статусами**:
  - Новое → В работе (при открытии оператором)
  - В работе → Новое (при новом сообщении от гражданина)
  - В работе → Завершено (автоматически через 5 мин. неактивности)
- 🔔 **Уведомления о новых сообщениях** с бейджами
- 🚪 **Простая авторизация** (без ограничений для тестирования)

### Для администраторов

- 📚 **Управление базой знаний** (CRUD операции)
- 🏷️ **Категоризация статей** и управление тегами
- 🔍 **Поиск и фильтрация** статей
- 📊 **Статистика** по статьям
- 🚪 **Защищённый доступ** (admin / admin)

### Интеллектуальные возможности

- 🧠 **ИИ-анализ обращений** (GigaChat API):
  - Определение приоритета (низкий, средний, высокий, критический)
  - Анализ тональности (позитивная, нейтральная, негативная)
  - Автоматическая категоризация
  - Извлечение ключевых слов
  
- 🔍 **Умный поиск в базе знаний**:
  - Поиск по всей БЗ (не ограничен категорией обращения)
  - Ранжирование релевантности (0-100+ баллов)
  - Кросс-категорийный поиск
  - Извлечение релевантных секций

- 🤖 **Генерация человекоподобных ответов**:
  - Персонализация под вопрос гражданина
  - Чистый текст без markdown
  - Профессиональное оформление
  - Контекстные ответы в диалоге
  - Умная фильтрация (игнорирует приветствия, благодарности и т.д.)

### Интеграция с Telegram

- 🤖 **Telegram бот** для приёма обращений:
  - Выбор категории через inline-кнопки
  - Описание проблемы текстом
  - Продолжение диалога с оператором
  - Автоматическое завершение сессии через 5 минут неактивности
  
- 💬 **Двусторонняя связь**:
  - Сообщения от граждан → Панель оператора
  - Ответы оператора → Telegram пользователю
  - Единая история переписки
  
- 🔄 **Управление сессиями**:
  - Автоматическое создание обращения
  - Хранение активной сессии пользователя
  - Автозавершение неактивных диалогов

### Компоненты системы

#### Frontend (3 приложения)

1. **User Interface** - Интерфейс для граждан
   - Форма подачи обращений
   - Чат с оператором
   - URL: `https://user-smartsupport.vadimevgrafov.ru`

2. **Operator Panel** - Панель для операторов
   - Управление обращениями
   - Чат с гражданами
   - ИИ-подсказки
   - URL: `https://operator-smartsupport.vadimevgrafov.ru`

3. **Admin Panel** - Панель для администраторов
   - Управление базой знаний
   - CRUD операции со статьями
   - URL: `https://admin-smartsupport.vadimevgrafov.ru`

#### Backend API

- **REST API** для обработки запросов
- **WebSocket** для чата в реальном времени (Socket.io)
- **Telegram Bot** интеграция (node-telegram-bot-api)
- **Авторизация** без ограничений (для тестирования)
- **Интеграция** с RabbitMQ для асинхронных задач

#### База данных (PostgreSQL)

- **Обращения** (appeals) с отслеживанием активности
  - `unread_operator_count` - счётчик непрочитанных сообщений
  - `last_activity_at` - время последней активности
  - `telegram_chat_id` - ID чата Telegram (если обращение из Telegram)
- **Пользователи** (users, operators)
- **База знаний** (knowledge_base, categories)
- **Чат** (chat_messages)
- **ИИ ответы** (ai_responses)

#### Очередь задач (RabbitMQ + Celery)

- **Анализ обращений** (analyze_appeal)
- **Генерация ответов** (generate_response)
- **Контекстные ответы** (contextual_response)
- **Автозавершение неактивных обращений** (auto_complete_appeals)
  - Запускается каждые 2 минуты (Celery Beat)
  - Завершает обращения без активности > 5 минут
  - Отправляет уведомление в Telegram (если обращение из Telegram)

#### Кэширование (Redis)

- Сессии пользователей Telegram
  - Выбранная категория
  - ID активного обращения
  - Состояние диалога
- Кэширование API запросов

## 🚀 Быстрый старт

### Предварительные требования

- Docker и Docker Compose
- Git
- Доступ к серверу (опционально)

### Локальная разработка

1. **Клонирование репозитория:**

```bash
git clone <repository-url>
cd it-support
```

2. **Настройка переменных окружения:**

```bash
# Единый файл для всех сервисов
cp .env.prod.example .env.prod

# Необходимо указать:
# - GIGACHAT_CLIENT_ID и GIGACHAT_CLIENT_SECRET
# - TELEGRAM_BOT_TOKEN (если используется)
# - DATABASE_URL, REDIS_URL, RABBITMQ_URL
# - Домены для nginx-proxy (VIRTUAL_HOST, LETSENCRYPT_HOST)
```

3. **Запуск с Docker Compose:**

```bash
# Запуск всех сервисов одной командой
docker-compose --env-file .env.prod up -d

# Просмотр логов всех сервисов
docker-compose --env-file .env.prod logs -f

# Просмотр логов конкретного сервиса
docker-compose --env-file .env.prod logs -f backend
docker-compose --env-file .env.prod logs -f celery-worker
docker-compose --env-file .env.prod logs -f celery-beat

# Остановка всех сервисов
docker-compose --env-file .env.prod down
```

4. **Проверка статуса:**

```bash
docker ps
```

## 💡 Использование

### Для граждан

#### Через Web-интерфейс

1. Откройте https://user-smartsupport.vadimevgrafov.ru
2. Заполните форму обращения:
   - ФИО
   - Телефон
   - Email (опционально)
   - Категория (выбор из списка)
   - Описание проблемы
3. Нажмите "Отправить обращение"
4. Автоматически открывается чат с оператором
5. Вы можете продолжать диалог в чате
6. Диалог автоматически завершается через 5 минут неактивности

#### Через Telegram

1. Найдите бота в Telegram (получите ссылку у администратора)

#### **Шаг 2: Автоматический ИИ-анализ (фоновые задачи)**

**А. Анализ приоритета и тональности (3-5 секунд)**
```python
# Celery task: analyze_appeal
1. GigaChat определяет приоритет:
   - critical: аварии, угроза жизни, полное отключение услуг
   - high: серьёзные проблемы, протечки, отсутствие отопления зимой
   - medium: качество услуг, начисления, документы
   - low: информационные запросы, консультации

2. GigaChat определяет тональность:
   - positive: благодарность, одобрение, похвала
   - neutral: вопрос, уточнение, описание без эмоций
   - negative: недовольство, жалоба, раздражение

3. Результаты сохраняются в БД (поля: priority, sentiment_type, ai_confidence)
```

**Б. Поиск в базе знаний и генерация рекомендации (5-8 секунд)**
```python
# Celery task: generate_response
1. Поиск статей ТОЛЬКО в категории обращения:
   - Выбранная категория определяет область поиска
   - Если статей в категории нет → confidence=0.2, сообщение оператору
   - Если статьи найдены → ранжирование по релевантности

2. GigaChat генерирует рекомендацию:
   - Использует ТОЛЬКО информацию из найденной статьи
   - Фильтрует "пустые" сообщения (приветствия, благодарности)
   - Генерирует структурированный ответ (2-5 предложений)
   - Профессиональный тон, избегает канцеляризмов

3. Результат сохраняется в таблице `ai_responses`:
   - suggested_text: текст рекомендации
   - confidence: уверенность (0.2-0.9)
   - sources: источники из БЗ
```

**⚠️ Важно:** ИИ работает **только в рамках выбранной категории**. Если пользователь выбрал "Мусор", а статей по этой категории нет — оператор увидит:
```

Рекомендуется:
1. Добавить статью в админ-панели базы знаний

```

- **high** — серьёзная проблема, создающая значительные неудобства (например, нет отопления, воды, света, уборки территории).
- **critical** — ситуация, представляющая угрозу жизни, здоровью, безопасности людей или имуществу (авария, пожар, затопление, угроза обрушения, замыкание, утечка газа и т.п.).

- "Произошёл прорыв трубы, вся квартира затоплена!" → critical
- "Где можно посмотреть график вывоза мусора?" → low
- "Во дворе повалено дерево, оно висит на проводах!" → critical

**Результат:**
- `low` — информационные запросы, консультации

### Как ИИ использует базу знаний

**Алгоритм поиска (файл:** `it-support/workers/tasks/generate_response.py`**)**

```python
def search_knowledge_base(category: str, text: str) -> list:
    """
    Ищет релевантные статьи ТОЛЬКО в указанной категории
    """
    1. Извлекает ключевые слова из вопроса (длина > 3 символов)
    2. Ищет совпадения в БД:
       - Категория = выбранная пользователем (обязательно!)
       - Ключевые слова в: title, content, tags
    3. Ранжирует результаты по релевантности:
       - Ключевое слово в title: +50 баллов
       - Ключевое слово в tags: +40 баллов
       - Ключевое слово в content: +10 баллов
    4. Возвращает топ-3 статьи с наивысшей релевантностью
    5. Если статей НЕТ:
       - Возвращает пустой список
       - ИИ генерирует сообщение: "⚠️ В базе знаний нет статей..."
```

**Пример:**
```
Вопрос: "Как часто менять счётчики воды?"
Категория: "Водоснабжение"

Шаг 1: Извлечение слов → ["менять", "счетчики", "воды"]
Шаг 2: Поиск в категории "Водоснабжение"
Шаг 3: Найдена статья "Замена счётчиков воды"
        - "счетчики" в tags: +40
        - "воды" в tags: +40
        - "менять" в content: +10
        Итого: 90 баллов

Шаг 4: GigaChat получает полную статью (первые 3000 символов)
Шаг 5: Генерирует ответ:
       "Счётчики холодной воды необходимо заменять каждые 6 лет,
        а горячей — каждые 4 года. Перед окончанием срока уведомите
        УК и согласуйте дату визита мастера."
```
